Sprint Review

Lowercase and Uppercase

I tested lowercase "a" and captial "A" to see what would happen as "A" is a menu option. When I entered "a" it printed unrecognised entry but when I entered "A" the program worked. I would need to fix this in the next version and validate the program so that it accepts both the lower and upper case variants.

Number not in list

When the value entered is not a number stated on the list specifically, the program fails. I need to make the program let the loop continue and have an error message when they do not enter a number stated on the list. It also does not allow the word version of the number so in my next version I should validate the program to allow this.

Negative number

If the user wants to eat fruit and then selects minus 10 as their amount it adds instead of takes away. I think I should add a message to make the program clearer or I should validate the program so the user cannot add negative numbers.

Eat fruit

If the user wants to eat more fruit than available the number of fruit available goes into the negatives. This needs to be validated as the user cannot eat negative fruit (fruit that is not there). I should also proably validate the program so that there is a max number of fruit you can add.
